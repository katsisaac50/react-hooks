# What are React Hooks?
<ul style={{textAlign: "left"}}>
  <Appear>
    <li>Functions for hooking into React state and lifecycle features</li>
    <li>There are some rules!</li>
    <li>Call them only at the top level</li>
    <li>...or from React function components (or your custom Hooks)</li>
  </Appear>
</ul>
---
## See what I mean (examples):

Built-in Hook <i>useEffect</i>
```jsx
useEffect(() => {
    const data = MockedApi.fetchData();
    data.then((res) => {
      res.forEach((e) => {
        if (e.name === friendName) {
          setIsPresent(true);
        }
      });
    });
  });
```
---
## See what I mean (examples):

Custom Hook <i>useFriendName</i>
```jsx
export default function useFriendName(friendName) {
  const [isPresent, setIsPresent] = useState(false);
  ...
  return isPresent;
}
```

# Why are they good thing?
<ul style={{textAlign: 'left'}}>
  <Appear>
    <li>Reusing logic - reuse your custom Hooks everywhere</li>
    <li>Giant components - avoid building huge, hard to maintain components</li>
    <li>Confusing classes - stop worrying about 'this' keyword and scope bindings</li>
  </Appear>
</ul>

# Conclusion
React Hooks are not really new feature that popped out just now. They are another (better ❓) way of doing React components that need to have state and/or lifecycle methods. Actually, they use the same internal logic that is being used currently by the class components. 

<i>To use them or not. This is the question that the future will give the best answer of.</i>
